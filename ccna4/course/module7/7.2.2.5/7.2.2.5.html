Capas de abstracción

Para ayudar a explicar cómo funciona la virtualización, es conveniente utilizar capas de abstracción en arquitecturas de PC. Un sistema de computación consta de las capas de abstracción siguientes, como se muestra en la Figura 1:

  • Servicios

  • SO

  • Firmware

  • Avanzado

En cada una de estas capas de abstracción, se utiliza algún tipo de código de programación como interfaz entre la capa inferior y la capa superior. Por ejemplo, el lenguaje de programación C suele usarse para programar el firmware que tiene acceso al hardware.

En la Figura 2, se muestra un ejemplo de virtualización. Un hipervisor se instala entre el firmware y el OS. El hipervisor puede admitir varias instancias de SO.

La figura 1 muestra un ejemplo de capas de abstracción de arquitectura informática. De abajo hacia arriba, tenemos el hardware (CPU, memoria, NIC, disco), sobre esta capa está el firmware (ROM), luego está el SO, luego están los servicios (Web, correo electrónico y servidores de archivos). La figura 2 muestra un ejemplo de capas de abstracción de arquitectura virtual. De abajo hacia arriba, tenemos el hardware (CPU, memoria, NIC, disco), sobre esta capa está el firmware (ROM), luego está el hipervisor, luego las tres instancias del sistema operativo todas en el mismo nivel y, por último, sobre esta capa están los servicios (servidor web, servidor de correo electrónico y servidor de archivos).
