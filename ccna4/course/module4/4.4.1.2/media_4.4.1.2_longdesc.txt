La ilustración es un diagrama de flujo que describe el A.C.L y el proceso de enrutamiento en un router. El primer cuadro indica que la trama llega a la interfaz del router. Ese cuadro lleva a un cuadro de decisión. El cuadro de decisión consulta si la dirección de capa 2 coincide o no. Si la respuesta es negativa, la trama se envía a la papelera. Si la respuesta es afirmativa, se consulta si las ACL están en la interfaz o no. Si la respuesta es negativa, el paquete se reenvía a una interfaz saliente. Si la respuesta es afirmativa, se debe tomar otra decisión para determinar si hay coincidencia con la lista de instrucciones de ACL. Si la respuesta es negativa, se aplica la instrucción de denegación predeterminada y el paquete se envía a la papelera. Si la respuesta es afirmativa, se debe decidir si permitir el paquete. Si el paquete no se permite, se lo envía a la papelera. Si el paquete se permite, se lo reenvía a una interfaz saliente. A continuación, se consulta si hay ACL en la interfaz. Si la respuesta es negativa, el paquete se envía a un dispositivo. Si la respuesta es afirmativa, se consulta si hay coincidencia con una de las instrucciones de la ACL. Si la respuesta es negativa, se aplica la instrucción de denegación predeterminada y el paquete se envía a la papelera. Si la respuesta es afirmativa, se consulta si se debe permitir el paquete. Si la respuesta es negativa, el paquete se descarta (se lo envía a una papelera). Si la respuesta es afirmativa, el paquete se envía al dispositivo.
