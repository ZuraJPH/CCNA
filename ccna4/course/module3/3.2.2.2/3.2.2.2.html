Verificación de PPPoE

Como se muestra en la Figura 1, el router del cliente se conecta al router ISP utilizando DSL. Ambos routers fueron configurados para PPPoE. El comando show ip interface brief se emite en R1 para verificar la dirección IPv4 asignada automáticamente a la interfaz de marcador por el router ISP.

Como se muestra en la figura 2, el comando show interface dialer en R1 verifica el encapsulamiento de MTU y PPP configurado en la interfaz del marcador.

En la figura 3, se muestra la tabla de routing en R1.

Tenga en cuenta que se instalaron dos rutas de host /32 para 10.0.0.0 en la tabla de routing R1. La primera ruta de host para la dirección asignada a la interfaz del marcador. La segunda ruta de host es la dirección IPv4 del ISP. La instalación de esas dos rutas de host es el comportamiento predeterminado para PPPoE.

Como se muestra en la figura 4, el comando show pppoe session se utiliza para mostrar información acerca de las sesiones de PPPoE actualmente activas. El resultado muestra las direcciones de Ethernet MAC locales y remotas de ambos routers. Las direcciones MAC de Ethernet se pueden verificar utilizando el comando show interfaces en cada router.

La figura 1 muestra la verificación de que la interfaz de marcador está activa. A la izquierda se ve una LAN conectada al router de cliente R1 etiquetada "G0/0". A la derecha de R1 mediante G0/1 sale un cable que va a un módem. G0/1 está etiquetado con ".1" y el módem está etiquetado con "10.1.3.0/24". El módem se conecta a otro módem dentro de la nube del ISP. El módem del ISP se conecta al router R2 mediante la interfaz G0/1 etiquetada ".2". R2 también está dentro de la nube del ISP. Se muestra la siguiente salida: R1# show ip interface brief. La salida son 6 columnas indicadas como interface, IP-address, OK? Method, status y protocol. La primera línea es la siguiente: Embeeded-service-engine0/0, unassigned, yes, unset, administratively down, down. La segunda línea es: gigabitEthernet0/0, unassigned, yes, unset, administratively down, down. La tercera línea es: gigabitEthernet0/1, unassigned, yes, unset, up, up. La cuarta línea es: serial0/0/0, unassigned, yes, unset, administratively down, down. La quinta línea es: serial0/0/1, unassigned, yes, unset, administratively down, down. La sexta línea está resaltada y muestra: dialer2, 10.1.3.1, yes, IPCP, up, up. La séptima línea es: virtual-access1, unassigned, yes, unset, up, up.
