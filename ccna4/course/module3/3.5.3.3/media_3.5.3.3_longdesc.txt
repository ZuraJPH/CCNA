La figura 1 tiene una tabla de dos columnas con un comando en la columna izquierda y la descripción en la columna derecha. El primer comando es show ip route, que se utiliza para verificar que las rutas anunciadas por el vecino BGP estén presentes en la tabla de routing IPv4. El siguiente comando es show ip bgp, que se utiliza para verificar que las redes IPv4 recibidas y anunciadas estén en la tabla de BGP. El último comando es show ip bgp summary, que se utiliza para verificar los vecinos BGP y demás información de BGP. La figura 2 muestra cómo el router perimetral en AS 65000 Empresa A recibió una ruta predeterminada a través del protocolo de routing BGP a través de la interfaz 209.165.201.1 del router vecino. La figura 3 muestra la tabla de BGP del router de la empresa A. Las columnas de la tabla son, de izquierda a derecha: red, siguiente salto, métrica, locprf, peso y ruta. La primera entrada es 0.0.0.0, 209.165.201.1 0, 0, 65001. La segunda entrada es 198.133.219.0/24 0.0.0.0, 0, 32768, i. La figura 4 muestra la salida del comando show ip bgp summary, que incluye el identificador de router de BGP para el router de la empresa A como 209.165.201.2, número local de AS 65000.
