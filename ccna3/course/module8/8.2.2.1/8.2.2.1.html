Habilitación de OSPF en las interfaces

El comando network determina qué interfaces participan en el proceso de routing para un área OSPFv2. Cualquier interfaz de un router que coincida con la dirección de red en el comando network está habilitada para enviar y recibir paquetes OSPF. El comando {119}network{120} también indica que la dirección de red (o de subred) para la interfaz está incluida en las actualizaciones de routing OSPF.

La sintaxis de comandos básica es network network-address wildcard-mask area area-id.

La sintaxis area area-id hace referencia al área OSPF. Al configurar OSPFv2 de área única, el comando network debe configurarse con el mismo valor area-id en todos los routers. Si bien se puede usar cualquier ID de área, es aconsejable utilizar una ID de área 0 con OSPFv2 de área única. Esta convención facilita la tarea si posteriormente se modifica la red para admitir OSPFv2 multiárea.
