Uso de una interfaz loopback como ID del router

Una ID de router también se puede asignar mediante una interfaz loopback.

La dirección IPv4 de la interfaz loopback se debe configurar con una máscara de subred de 32 bits (255.255.255.255). Esto crea una ruta de host. Una ruta de host de 32 bits no se anuncia como ruta a otros routers OSPF.

En el ejemplo de la ilustración, se muestra cómo configurar una interfaz loopback con una ruta de host en el R1. El R1 usa la ruta de host como ID del router, suponiendo que no se configuró ninguna ID de router de manera explícita o que no se obtuvo anteriormente.

Nota: El comando router-id es el método preferido. Sin embargo, algunas versiones anteriores de IOS no reconocen el comando router-id; por lo tanto, la mejor forma de establecer la ID del router en esos routers es mediante una interfaz de loopback.

En la figura uno, se muestran los comandos utilizados para configurar una interfaz de loopback en un router para que funcione como la ID del router. La figura comienza en el modo de configuración global. El primer comando es interface loopback cero, que avanza a la configuración de la interfaz. El siguiente comando es ip address uno punto uno punto uno punto uno 255 punto 255 punto 255 punto 255. Este comando establece la dirección IP. Esta es ahora la ID del router.
