Topología de la red OSPFv3

En la figura 1, se muestra la topología de la red que se utiliza para configurar OSPFv3.

En la figura 2, se muestra el routing de unidifusión IPv6 y la configuración de las direcciones de unidifusión global del R1, como se identifican en la topología de referencia. Suponga que las interfaces del R2 y el R3 también se configuraron con sus direcciones de unidifusión global, como se identifica en la topología mencionada.

En esta topología, ninguno de los routers tiene direcciones IPv4 configuradas. Una red con las interfaces del router configuradas con direcciones IPv4 e IPv6 se denomina “dual-stack”. Una red dual-stack puede tener OSPFv2 y OSPFv3 habilitados de manera simultánea.

En la figura 3, se muestran los pasos para configurar OSPFv3 básico en un área única.

En la figura uno, se muestra la topología de referencia de OSPF con asignación de direcciones IPv6 en lugar de las direcciones IPv4 utilizadas anteriormente con OSPFv2. En la figura dos, se muestran los comandos necesarios para configurar la dirección y el OSPFv3 en el router R1. La figura comienza en el modo de configuración global y el primer comando es ipv6 unicast guion routing. El siguiente paso consiste en pasar a una interfaz específica, y el comando es interface gigabit ethernet cero barra diagonal cero. El siguiente comando es description R1 LAN. El siguiente comando es ipv6 address 2 0 0 1 dos puntos D B 8 dos puntos C A F E dos puntos 1 dos puntos dos puntos 1 barra diagonal 64. Este comando configura la dirección IPv6 en la interfaz. Luego, la interfaz se activa con el comando "no shut". La figura continúa con la primera interfaz serial. El primer comando es interface serial cero barra diagonal cero barra diagonal cero. El siguiente comando es description link to R.2. El siguiente comando es ipv6 address 2 0 0 1 dos puntos D B 8 dos puntos C A F E dos puntos A 0 0 1 dos puntos dos puntos 1 barra diagonal 64. Este comando configura la dirección IPv6 en la interfaz. Luego, la interfaz se activa con el comando "no shut". La figura continúa con la segunda interfaz serial. El primer comando es interface serial cero barra diagonal cero barra diagonal uno. El siguiente comando es description link to R.3. El siguiente comando es ipv6 address 2 0 0 1 dos puntos D B 8 dos puntos C A F E dos puntos A 0 0 3 dos puntos dos puntos 1 barra diagonal 64. Este comando configura la dirección IPv6 en la interfaz. Luego, la interfaz se activa con el comando "no shut". En la figura tres, se indican los seis pasos para configurar OSPF versión 3. El paso uno es habilitar el routing de unidifusión de IPv6 con el comando ipv6 unicast guion routing. El paso dos es opcional. En el paso dos, se configuran las direcciones de enlace local. El paso tres consiste en configurar una ID de router de 32 bits en el modo de configuración de router de OSPF versión 3 mediante el comando router guion id. El paso cuatro consiste en configurar datos específicos de routing opcionales, como el ajuste del ancho de banda de referencia. El paso cinco es opcional. En el paso cinco, se realiza la configuración específica de la interfaz OSPF versión 3, como el ancho de banda de la interfaz. El paso seis consiste en habilitar el routing IPv6. con el comando ipv6 ospf area.
